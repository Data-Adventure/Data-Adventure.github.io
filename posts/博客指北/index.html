prepare two repositories on github

why do we need two repositories ?

  1. we don’t need to use two git repositories to manage files
  2. once git push, Once the git push can be completed with the github action to ：
    1. generate a static file
    2. Deploy and move static files to appropriate locations
  3. keep your website files from disclosing personal information

！build two empty repositories

  • repository name XXX.github.io
    • XXX is your github user name
    • public
    • store your website static files
  • repository name name is yyy, yyy is your favor name
    • private

# Sample workflow for building and deploying a Hugo site to GitHub Pages
name: Deploy Hugo site to Pages

on:
  # Runs on pushes targeting the default branch
  push:
    branches:
      - main # your git is branch's name

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

jobs:
  build-deploy:
    runs-on: ubuntu-22.04
    # runs-on: macos-latest # 本地编译的是macos
    steps:
      - name: Check out source
        uses: actions/checkout@v2
        with:
          submodules: true
          fetch-depth: 0

      - name: Setup hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: "latest"
          extended: true # 设置是否需要 extended 版本

      - name: Build
        run: hugo

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.PERSONAL_TOKEN }} # 另外还支持 deploy_token 和 github_token
          external_repository: Data-Adventure/Data-Adventure.github.io # 修改为你的 GitHub Pages 仓库
          publish_dir: ./public
          #          keep_files: false
          publish_branch: main
          # 如果使用自定义域名，还需要添加下面一行配置
          # cname: www.goodfun.me

#!/bin/bash

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[0;33m'
BLUE='\033[0;34m'
NC='\033[0m'

print_msg() {
    COLOR=$1
    MSG=$2
    echo  "${COLOR}${MSG}${NC}"
}
# 读取修改内容
print_msg "$GREEN" "What's your update information ?:"
read -r update

print_msg "$GREEN" "Have you confirmed ?:"
read -r confirmed
if [ "$confirmed" == 'n' ];then
    print_msg "$RED" "Submit terminate!"
    exit 1
else 
    print_msg "$YELLOW" "Confirmed!"
    print_msg "$BLUE" "Your update is $update, accepted!"
    
    # 设置提交说明，格式为 Site updated: 2006-01-02 15:04:05
    time=$(date "+%Y-%m-%d %H:%M:%S")
    commit="Site updated:"$time

    # 生成
    hugo -DF
    # 输出信息
    git add .
    git commit -m "$commit"
    # 推送到source分支上
    git push
    print_msg "$BLUE" "All done!"
fi
